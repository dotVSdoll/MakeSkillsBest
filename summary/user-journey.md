# 用户旅程

## 全景流程图

```
┌─────────────┐   ┌──────────────────────┐   ┌──────────────────┐
│ ① 安装      │ → │ ② 运行 Gardener     │ → │ ③ 进入花园      │
│ (2 步搞定)  │   │ (/gardener 一条命令) │   │ (Pygame 自动弹窗)│
└─────────────┘   └──────────────────────┘   └──────────────────┘
                                                      │
                                                      ├── 看健康度
                                                      ├── 查问题
                                                      ├── 改规则
                                                      └── 设调度
```

## 第一步：安装

```bash
git clone https://github.com/dotVSdoll/little-gardener.git
```

然后打开项目 → **session-start hook 自动运行**：
- 检测 Python 是否安装 ✅
- 检测 pygame 是否安装 → 缺则 `pip install pygame` ✅
- 检测 sprites 是否存在 ✅

用户感知：打开项目直接能用，不用手动装任何东西。

## 第二步：运行

```bash
/gardener "检查项目上下文文件健康度"
```

Agent 自动执行：

| 用户看到 | 后台发生 |
|----------|---------|
| "🔍 正在扫描花园..." | scanner.py 遍历 .claude/ 所有 .md |
| "🩺 发现 3 个问题" | analyser.py 检测到 stale + contradiction |
| "📋 建议：审查 CLAUDE.md | Plan 生成建议 |
| "🔧 是否修剪？(y/n)" | Act 阶段等待确认 |
| "✅ 健康分 62 → 78" | Verify 重新检查 |
| "📝 已记录" | Learn 写入 memory |

## 第四步：持续维护

| 场景 | 用户操作 |
|------|---------|
| 想看花园状态 | `/gardener`（或在项目目录打开 garden） |
| 调整规则 | 打开花园 → 按 S → 修改配置 |
| 设定时任务 | 设置面板 → 调度 → 选 cron |
| 深层修剪 | `/gardener 修剪花园 --apply` |

## 安装模型对比

| 方法 | 好处 |
|------|------|
| Claude Code hook | 零手动操作，自动检测 |
| 传统 pip install | 可预测，其他 Agent 工具也适用 |
| 推荐组合 | hook 做初始化 + pip 做 fallback |
